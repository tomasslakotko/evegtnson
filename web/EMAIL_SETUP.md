# Email Integration Setup

## Resend Setup (БЕЗ ДОМЕНА - для тестирования)

**Хорошая новость**: Для тестирования домен НЕ требуется! Можно использовать `onboarding@resend.dev`.

1. **Создайте аккаунт Resend**: Перейдите на https://resend.com и зарегистрируйтесь (бесплатный план: 3000 писем/месяц, 100 писем/день)

2. **Получите API ключ**:
   - Перейдите на https://resend.com/api-keys
   - Создайте новый API ключ
   - Скопируйте ключ

3. **Добавьте в `.env` файл**:
```bash
RESEND_API_KEY=re_xxxxxxxxxxxxx
# RESEND_FROM_EMAIL можно не указывать - будет использоваться onboarding@resend.dev
```

**Важно**: Если вы не укажете `RESEND_FROM_EMAIL`, система автоматически использует `onboarding@resend.dev`, который работает без домена для тестирования.

## Для продакшена (когда будет домен)

Когда у вас появится домен:
1. В Resend перейдите в "Domains"
2. Добавьте ваш домен
3. Добавьте DNS записи, которые Resend предоставит
4. После верификации обновите `.env`:
```bash
RESEND_FROM_EMAIL=noreply@yourdomain.com
```

## Email Notifications

Система отправляет email уведомления для:

1. **Подтверждение бронирования** - При создании бронирования
2. **Изменение времени** - При изменении времени встречи
3. **Обновление информации** - При обновлении деталей бронирования
4. **Отмена бронирования** - При отмене встречи
5. **Ссылка на встречу** - При назначении ссылки на встречу

Все письма отправляются на русском языке и включают:
- Детали встречи (название, дата, время)
- Информацию о хосте
- Ссылку на встречу (если доступна)
- Тип локации

## Тестирование

Для тестирования email функциональности:
1. Убедитесь, что `RESEND_API_KEY` установлен в `.env`
2. Создайте бронирование
3. Проверьте почту участника
4. Проверьте логи сервера на наличие ошибок (email отправляются асинхронно и не прерывают процесс бронирования при ошибке)

## Альтернативы (если Resend не подходит)

Если по каким-то причинам Resend не работает, можно использовать:
- **SendGrid** (бесплатно до 100 писем/день) - требует верификации email
- **Mailgun** (бесплатно до 5000 писем/месяц) - требует домен
- **SMTP через Gmail** (бесплатно, но ограничения) - требует настройки App Password
