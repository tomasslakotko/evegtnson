# Важно: Нужна точная строка из Supabase!

## Проблема
Ошибка "Tenant or user not found" означает, что формат connection string неправильный.

## ✅ Решение

### Скопируйте ТОЧНУЮ строку из Supabase:

1. В Supabase на вкладке "Connection String"
2. Убедитесь, что выбрано:
   - Type: URI
   - Source: Primary Database
   - Method: Session pooler
3. **Скопируйте ВСЮ строку** которая отображается (начинается с `postgresql://...`)
4. **Вставьте её сюда** - я обновлю .env файл

### Или сделайте скриншот
Сделайте скриншот строки connection string из Supabase, и я помогу правильно её использовать.
